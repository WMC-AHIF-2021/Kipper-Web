  • Kipper Datatypes

    As previously shown in the docs page Variables, a variable always has a type that defines how you can interact with it. This also means that you often can not mix variables of different data types together, as they fundamentally represent different things.

    What is a datatype?

    A data type defines the type of value saved in a variable or constant. A variable with the datatype num, for example, can only contain numbers. A variable with an str datatype can contain text, symbols or numbers, but saves them as a text. This makes them for example impossible to use for calculations.

    Remember!

    You may NEVER mix data types or do incompatible interactions with them!

    List of data types in Kipper

    • Numeric Value - num

      Represents a number (both floating point and integer) in the Kipper language. This datatype may always be used with the standard mathematical expressions.

      Examples
      // âœ“ Simple integer number
var var1: num = 400;

// âœ“ Simple floating point number
var var2: num = 0.43493;

// âœ“ Calculations using floating point 'num' and integer 'num'
var var3: num = var2 + var1; // -> 400.43493

// âœ“ Plus
var var1: num = 400.3 + 26.3; // -> 426.6

// âœ“ Minus
var var2: num = 87 - 2.5; // -> 84.5

// âœ“ Multiply
var var2: num = 2.4 * 5; // -> 12

// âœ“ Divide
var var2: num = 25 / 4; // -> 6.25

// âœ“ Power to
var var2: num = 2 ** 8; // -> 256

// âœ“ Rest of Divide
var var2: num = 51 % 10; // -> 1

      Important

      Remember that Kipper will interpret mathematical calculations as defined in the standard math conventions. (Multiplications, divisions and power-to operations have a higher priority than addictions and subtractions) 

      // âœ“ Standard math rules apply
var result: num = 4 + 4 * 5; // -> 24

      Additionally, to the default Base-10 values, you may also use the following with num: 

      // âœ“ Bytes Support
var bytes: num = 0b11111010; // -> 250

// âœ“ Hex-Decimal Number Support
var hex: num = 0xFFA2; // -> 65442

// âœ“ Octal Number Support
var octal: num = 0o347; // -> 231

    • Char Value - char

      A char-value is a simple UTF-16 character, which may represent any Unicode character that is defined in the UTF-16 standard. To define a simple character, you use the '' syntax, hinting a single character.

      Examples
      // âœ“ Simple UTF-16 Character
var var6: char = '4';

// âœ“ Single character UTF-16 character
var var7: char = "Æ›";

// X Char may not be empty
var var8: char = '';

// X Char may only be a single character
var var9: char = 'This is obviously more than a single character';

// X Char has to be a string and "" signalises a string!
var var10: char = "x";

// X No direct calculations between 'char' and 'num' without conversion
var var11: char = '8'; // -> 8
var var12: num = 48; // -> 0
var result: char = var10 + var11; // -> Incompatible types 'char' and 'num'

// X No usage of emojis that use multi-character UTF-16 sequences like ðŸ’˜ (U+1F498 / 128152), which are bigger than the
// allowed range of 65535. For these use 'str' instead, as they require multiple characters!
var var13: char = 'ðŸ’˜'; // -> X Char may only be a single character

// âœ“ Multi character UTF-16 characters in strings
var var14: str = "ðŸ’˜";

    • Boolean Value - bool

      The boolean is a simple binary value that may be either True or False. The value can be also represented as 0 (False) and 1 (True).

      Examples
      // âœ“ Simple bool value using 'True' and 'False'

var var16: bool = True;

// âœ“ Simple comparison

var var18: num = 48;
var is_equal: bool = var10 <= var11; // 3 smaller than 48 -> True

// âœ“ Boolean calculations with numbers

    • Void Value - void

      The void type represents a value that is nothing and may only be void. This is usually used for functions to hint they have no return. Otherwise, this data type is practically useless, and it should be avoided for variable usage!

      Examples
      // âœ“ Creating a void variable - This may only be 'void'
var var21: void = void;

// X May not set a 'void' variable to anything except 'void'
var var22: void = 4;

// âœ“ Returning void for a function - Returns 'void' per default
def func1() -> void {
  return;
}

// âœ“ Returning void for a function manually
def func2() -> void {
  return void;
}

// âœ“ Assigning void return to variable of type 'void'
var var23: void = func2();

    • List Value - list<type>

      The list<type> data type is a unique data type, as it does not represent itself a value, but rather a sequence of other data types. Therefore, a list has also a length, and indexes for each item, which you may use to access them using the following syntax: 

      VAR[INDEX]
      Examples
      // âœ“ Creating a simple list
var var24: list<num> = [ 2, 3, 4 ];

// âœ“ Accessing item of value per index (0 = first item)
var var25: list<num> = [ 2, 3, 4 ];
var item_of_list: num = var26[2]; // -> 4

// âœ“ Accessing the length of the list using 'len()'
var len_of_list: num = len(var26); // -> 3

// âœ“ Accessing the last item of the list using 'last()'
var len_of_list: num = last(var26); // -> 4

// âœ“ Accessing the first item of the list using 'first()'
var len_of_list: num = first(var26); // -> 2

// X May not set a list to a single value
var var26: list<num> = 2;

// X May not set item of list to an invalid type
var var27: list<num> = [ "string" ];

// X May not convert list to another type as a whole!
var var28: list<str> = [ "99", "1893", "4" ];
var var29: list<num> = var27 as list<num>; // -> Invalid conversion
