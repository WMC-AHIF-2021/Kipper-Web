    Welcome to the Quickstart Guide for the Kipper programming language! Here you will find a concise guide on how to install Kipper and start using it for your own projects.

    Before using Kipper make sure your system or browser is supported! View support list here!

    How to use Kipper?

    Since Kipper is based on TypeScript and therefore also JavaScript, there are several options to using Kipper:

    1. Running Kipper in the browser using native JavaScript.
    2. Running Kipper using Node.js in your terminal.
    3. Including Kipper as a library into your own Node.JS programs.

    Option 1. - Setting up Kipper for the browser

    For running Kipper in the browser, you will have to include:

    1. The primary kipper-standalone.min.js file, which provides the kipper source code compatible for modern browsers.
    2. The babel typescript compiler, which allows native browser transpiling and code generation. This is required as Kipper is not a language that compiles directly to binary code, but uses another language as another level of abstraction.

    Simple snippet to include into your HTML file:

    <!-- Babel dependency -->
<script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>

<!-- Kipper -->
<script src="https://cdn.jsdelivr.net/npm/@kipper/base@latest/kipper-standalone.min.js"></script>

    After including Kipper into your HTML file, you can access the library using the identifier Kipper, which will be globally available in your file.

    Example code

    <script type="module">
  // Define your own logger and compiler, which will handle the compilation
  const logger = new Kipper.KipperLogger((level, msg) => {
    console.log(`[${Kipper.getLogLevelString(level)}] ${msg}`);
  });

  // Define your own compiler with your wanted configuration
  const compiler = new Kipper.KipperCompiler(logger, { });

  // Compile the code to Typescript
  // Top-level await ref: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/await#top_level_await
  const result = (await compiler.compile(`call print("Hello world!");`)).write();

  // Transpile the TS code into JS
  const jsCode = Babel.transform(result, { filename: "kipper-web-script.ts", presets: ["env", "typescript"] });

  // Finally, run your program
  eval(jsCode.code);
</script>

    Option 2. - Setting up Kipper for the Terminal using Node.js

    To use Kipper in a terminal, you will have to install the package @kipper/cli, which provides the CLI implementation that allows you to use Kipper over the terminal. 

    "dependencies": {
  ...
  "@kipper/cli": "^0.2.1"
}
/* or */
"devDependencies": {
  ...
  "@kipper/cli": "^0.2.1"
}

    Option 3. - Setting up Kipper as a library

    To set up Kipper for your personal code, simply add @kipper/base to your dependencies or development dependencies in your package.json file. 

    "dependencies": {
  ...
  "@kipper/base": "^0.2.1"
}
/* or */
"devDependencies": {
  ...
  "@kipper/base": "^0.2.1"
}

    If you intend to only try out Kipper, you can also simply install Kipper globally for the time you are using it, like this: 

    npm install -g @kipper/base

    Important

    If possible avoid installing Kipper globally if you intend to use it inside your own projects, as it can cause issues with locally installed versions (node project dependencies). For a localised and project-dependent kipper installation add kipper to your dependencies.
